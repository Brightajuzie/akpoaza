# FixMart Backend вАФ Render.com Deployment Guide

> **Target platform**: [Render.com](https://render.com) Web Service (Node.js)  
> **Database**: Supabase PostgreSQL (pooled + direct connections)

---

## Prerequisites

| Tool | Version |
|------|---------|
| Node.js | вЙ• 20.x |
| npm | вЙ• 10.x |
| Prisma CLI | bundled in devDeps |

Make sure the following are ready before deploying:
- A **Supabase** project with the schema migrated (`prisma migrate deploy`)
- All third-party API keys (Stripe, Paystack, Dojah, etc.)
- A Render account with a **Web Service** or **Blueprint** wired to this repository

---

## Step 1 вАФ Database (Supabase)

1. Open your Supabase project вЖТ **Settings вАЇ Database**.
2. Copy both URLs:
   - **Pooling** (mode = *Transaction*, port `6543`) вЖТ `DATABASE_URL`
   - **Direct** (port `5432`) вЖТ `DIRECT_URL`
3. Append `?pgbouncer=true` to `DATABASE_URL` if it is not already present.
4. Run migrations once from your local machine (or a Render one-off job):

```bash
# From backend/
DATABASE_URL="<pooled>" DIRECT_URL="<direct>" npx prisma migrate deploy
```

> [!WARNING]
> Never run `prisma migrate dev` against the production database. Use `prisma migrate deploy` only.

## Step 2 вАФ Create the Render Web Service

### Option A вАФ Blueprint (recommended)

Render detects `render.yaml` automatically.

1. Go to **Render Dashboard вАЇ New вАЇ Blueprint**.
2. Connect your GitHub/GitLab repository.
3. Render reads `backend/render.yaml` and creates the service.
4. Fill in the `sync: false` env vars in the **Environment** tab (see Step 3).
5. Click **Apply**.

### Option B вАФ Manual Service

1. **New вАЇ Web Service** вЖТ connect the repo.
2. Set **Root Directory** to `backend` (if your repo root is the `fixmart/` monorepo).
3. Set **Build Command**: `npm ci && npm run build`
4. Set **Start Command**: `npm start`
5. Set **Node version**: `20` (or use the `.node-version` file).
6. Add env vars (see Step 3).

## Step 4 вАФ Stripe Webhooks

1. In the [Stripe Dashboard](https://dashboard.stripe.com/webhooks) вЖТ **Add endpoint**.
2. URL: `https://<your-render-url>/api/payments/stripe-webhook`
3. Events to subscribe: `payment_intent.succeeded`, `payment_intent.payment_failed`
4. Copy the **Signing secret** (`whsec_вА¶`) вЖТ set as `STRIPE_WEBHOOK_SECRET` on Render.

---

## Step 5 вАФ Paystack Webhooks

1. In [Paystack Dashboard](https://dashboard.paystack.com/#/settings/developer) вЖТ **Webhooks**.
2. URL: `https://<your-render-url>/api/payments/paystack-webhook`
3. No signature secret needed вАФ Paystack uses your `PAYSTACK_SECRET_KEY` to sign.

## Step 6 вАФ Verify the Deployment

After the first successful deploy:

```bash
# Health check
curl https://<your-render-url>/health
# Expected: {"status":"ok","timestamp":"..."}
```

Check the Render logs for any startup errors. Common issues:
- `DIRECT_URL` not set вЖТ Prisma crashes on boot
- `JWT_SECRET` is the dummy fallback вЖТ warning logged (not a crash)
- TypeScript compile error вЖТ check the build log

## File Storage вЪ†пЄП

> [!WARNING]
> Render's filesystem is **ephemeral**. Any files written to `uploads/` are wiped on every deploy or restart. This means all user-uploaded images will be lost.
>
> **For production**, migrate to cloud storage before going live:
> - [Cloudinary](https://cloudinary.com) вАФ easiest to integrate with the existing `upload.ts` route
> - [AWS S3](https://aws.amazon.com/s3/) + CloudFront CDN
> - [Supabase Storage](https://supabase.com/storage) вАФ already in your stack
